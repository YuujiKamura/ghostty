# Fork Isolation Audit — 2026-04-27

Companion to issue #238. Forward-looking defense (rule + lint) lives in
`AGENTS.md` Operating Rule #7, `docs/apprt-contract.md`, and
`tools/lint-fork-isolation.sh`. This document captures the backward-looking
audit: what already lives outside fork-owned paths, why it sprawled, and
how each cluster maps to a refactor backlog.

## Meta-observation

Phase 2 pilot (commits `4e661fcab` + `3ff0756d1`, landed 2026-04-27) was
dispatched BEFORE the apprt contract was articulated, and demonstrates the
failure mode this audit addresses: even a "contract-aware refactor" effort
sprawled into upstream-shared paths because the contract wasn't yet
codified. The audit treats Phase 2 as a recoverable misstep — relocate
before Phase 2.3 to establish the correct pattern.

The irony of writing the contract while violating it in parallel is itself
evidence for why the lint script and the AGENTS.md rule must exist.

## Category 1: MOVABLE — extract to `apprt/winui3/`

55 files contain Win32/WinUI3/D3D11/DirectWrite/HLSL/fontconfig identifiers
in their fork diff. They are not by definition movable — the heuristic
flags candidates for review — but most represent in-place modifications
that the wrapper patterns in `docs/apprt-contract.md` could have absorbed.

### High-priority (P0) entries

#### `src/datastruct/bounded_mailbox.zig` (NEW, 741 lines, Phase 2 pilot, #232)

- **Introduced**: commits `4e661fcab` (Phase 2.1) + `3ff0756d1` (Phase 2.2),
  landed 2026-04-27.
- **WHY sprawled**: Phase 2 added `BoundedMailbox<T,N,timeout>` directly
  under `src/datastruct/` because it was framed as a generic stdlib type.
  At the time the fork-isolation discipline had not been codified, so the
  wrapper-first reflex was missing.
- **Status**: PARTIAL ALIGNMENT — the type itself is the right shape
  (bounded + timeout-aware mailbox), but it landed in the wrong location.
- **Refactor**:
  - Current: `src/datastruct/bounded_mailbox.zig`
  - Target:  `src/apprt/winui3/safe_mailbox.zig` (winui3-local wrapper of
    upstream's `BlockingQueue`).
  - Re-export from the new location; update consumers in
    `src/renderer/Thread.zig` and `src/termio/*` to use the wrapper via
    policy injection instead of swapping upstream's `renderer.Thread.Mailbox`
    wholesale. `src/apprt/embedded.zig` should be reverted (we don't use
    that apprt — same shape as the #224 contamination).
- **Effort**: 4-6h (relocate + update consumers + revert other-apprt edits).
- **Priority**: **P0** — sets the contract-aware pattern for Phase 2.3 and
  every future deadlock fix. Landing this BEFORE Phase 2.3 sweep prevents
  Phase 2.3 from compounding the violation.

#### `src/datastruct/blocking_queue.zig` — `shutdown()` method (#220)

- **Introduced**: deadlock cluster commit (single commit).
- **WHY sprawled**: needed cooperative shutdown for CP server; added the
  method directly to upstream's stdlib type because no wrapper existed.
- **Refactor**: subsumed by the BoundedMailbox relocation above. Once
  `apprt/winui3/safe_mailbox.zig` exists, `shutdown()` semantics live
  there; upstream's `BlockingQueue` reverts to its untouched form.
- **Effort**: rolled into the BoundedMailbox 4-6h estimate.
- **Priority**: P0 (linked to the entry above).

#### `src/Surface.zig` — focus mailbox push policy (#218 family)

- **Introduced**: deadlock cluster fixes plus 13 callsite migrations from
  Phase 2.2.
- **WHY sprawled**: `Surface.zig` is upstream-shared but the
  `.forever = {}` push pattern was tuned in-place to fit Win32 timing.
- **Refactor**: extract the mailbox push policy into a generic / comptime
  parameter on Surface (contract extension, last-resort wrapper pattern
  in `docs/apprt-contract.md`). `apprt/winui3/` injects the bounded
  policy; other apprts continue to inject `.forever = {}`.
- **Effort**: 6-10h (touches every Surface mailbox callsite).
- **Priority**: P1 — high merge friction every upstream Surface change.

#### `src/Command.zig` — Win32 wait timeout (#221)

- **Introduced**: 1 commit (deadlock cluster).
- **WHY sprawled**: needed bounded `WaitForSingleObject` semantics; added
  to `Command.zig` instead of wrapping.
- **Refactor**: extract Win32-specific wait into `src/apprt/win32/spawn.zig`
  per pattern 2 in `docs/apprt-contract.md`.
- **Effort**: 3-5h.
- **Priority**: P1.
